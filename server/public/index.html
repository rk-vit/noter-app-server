Notes API Documentation

This is the API documentation for the Notes API. You can interact with the API using the following endpoints.

GET /api

Description: Fetch all the notes stored in the application.

Request: No request body is required.

GET http://localhost:3000/api

Response: A list of all notes in JSON format.

{
    "notes": [
        {
            "id": 1,
            "title": "Note 1",
            "text": "This is the first note."
        },
        {
            "id": 2,
            "title": "Note 2",
            "text": "This is the second note."
        }
    ]
}

POST /api

Description: Create a new note and store it in the application.

Request Body: You must send a JSON body containing the title and text of the note.

{
    "title": "New Note",
    "text": "This is a new note created via API."
}

Response: A success message with a status code 201.

HTTP/1.1 201 Created

DELETE /api

Description: Delete an existing note from the application based on its ID.

Request Body: You must send a JSON body containing the key (ID) of the note to be deleted.

{
    "key": 1
}

Response: A success message with a status code 204 (No Content), indicating the note was successfully deleted.

HTTP/1.1 204 No Content

Notes:

  • All requests should be made to http://localhost:3000/api
  • The POST request requires a JSON body with title and text for creating a new note.
  • The DELETE request requires a JSON body with a key field representing the note ID to be deleted.
